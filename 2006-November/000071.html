I too have a simple scripting system for batches of backups! :D

Mine is designed to have a number of users (i.e. using the mysql
backend) and at run time, can filter and only run the backups for a
certain user. It also keeps all repos in the format of
/path/to/repos/user/backupname which should make things work well for
rdiffWeb.

The input file is in the format of
user,source,destination,switches

If your interested in it, head to
<http://members.plug.org.au/~linuxalien/dokuwiki/scripts:rdiff-backup_job_runner>

Hopefully it'll help someone else!

Tim

$ ./rdiff-backup_job_runner --help

        rdiff-backup_job_runner ver. 0.0.1
        This is a program to Run a set of rdiff-backup's.
        It uses a comma delimited config file of the format:
                username,src,dst,switches
        The switches are optional and can be left blank

        Usage: rdiff-backup_job_runner [-h | --help] [-s] [-x "forced

        [-o "extra rdiff-backup options"] [-u filteruser] [-f config_file]

        Options:

        -h, --help      Display this help message and exit.
        -s                      Skip last backup time check
        -x "options"    Force a set of rdiff-backup options
        -f filename     Use a different config file to the default
        -o "options"    Append rdiff-backup options to default set
        -u user         Only run backups for this user
