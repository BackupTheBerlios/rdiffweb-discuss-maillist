data. So far it take less then a second to sort 11000 items in Chrome
browser vs 12 secs before the modification. The biggest modification is the
use of TimSort algorithm instead of the default JavaScript sort. The
modification is satisfying on the client side, but the server side is still
slow.

I'm in process to find where is the bottleneck on the server side.
