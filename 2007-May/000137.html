Here is a copy/paste from an earlier email that I sent about the admin 
page. Right now, it is a really nasty mess. I'd like to fix it sometime, 
but I'm rather busy at the moment. I'll try to take a look at patch 
soon, but no promises.

I've been wanting to rewrite the admin functionality for some time. 
Things that would be nice to accomplish:
* Allowing users to change their own passwords
* Allowing users to update their list of repositories (i.e. spider for 
new ones.)
* Allowing users to administrate their root folders, for example add 
sub-users, and those sub-users could add sub-sub-users, etc. This could 
be useful for when administers back up a server that contains multiple 
home directories, and want users to be able to restore files from their 
home folders, without having access to the whole backup.
* Providing a nice gui in general. The current admin interface is horrible.

Any thoughts on this? Anybody? Particularly the point about allowing 
sub-users...
