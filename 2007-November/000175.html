I thought it might be helpful to let everybody know where the current 
plans for rdiffWeb are headed. The major changes I'd like to make include:
* Switch the database backend to use sqlite instead of mysql and the 
file backend. This will remove the need for setting up a mysql server, 
and will remove the need to continue forking database backends (which 
currently adds a lot of maintenance overhead.)
* Switching to cherrypy 3 (to get built-in https)
* Web-based initial configuration. This should clear the way to get 
better distro packages.
