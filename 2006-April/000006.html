One option would be to have a query string param with the username.  
Would this work for Thunderbird?  Not sure I really like that though, bc 
that exposes the username for whoever happens to be listening on the 
connection (web proxies, etc).

Allowing the user to have multiple roots would break a lot of the 
assumptions we have in rdiffWeb, but that doesn't mean we couldn't do 
it.  I am having a little trouble coming up with a scenario where many 
people would want to have multiple roots, though.  I think it could get 
confusing pretty quickly.  What happens if you have a backup with the 
same name in two different roots?
