Hello.

Sorry, but I have very limited resources at the moment, so I can't tell why 
its not working, but I do have the same problem under CygWIN.

I have been able to devised a horrible quick fix. It could cause other 
problems if you uses some rare characters in filenames. Specifically, look 
for a file called chars_to_quote in the rdiff-backup-data directory in your 
backup location. If you use any character in that file then this will 
probably break something else. Caveat emptor.

This fix is:
Find the file called librdiff.py. It should be in 
/usr/lib/python2.5/site-packages/rdiffWeb

Line 138 of the file should be:
return self.quoteRegex.sub(self.getQuotedChar, unQuotedPath)
Replace it with:
return unQuotedPath

Please let me know whether this fixes your problem. Whoever eventually finds 
the correct solution may find that useful.

Best of luck.
