Try pointing your users to the directory *above* the backup locaiton.
so
serverx <password> /home/rdiffbackup/backups

Initially, that will give user "serverx" access to both backups if it works, but will at least show the solution with minimal effort.

Change your paths to
/home/rdiffbackup/server1/backup
/home/rdiffbackup/server2/backup
and adjust the users accordingly to get back your original setup.

I've installed rdiffweb because it looks like a convenient and simple client to the rdiff-backup data to easily parse through and restore some files, and get statuses of backups... and I can monitor them from any location.... this sounds great.  The only problem is that rdiffweb is not finding my rdiff-backup directrories.  I'm using rdiffweb 0.6.3

/home/rdiffbackup/backups/server1 and
/home/rdiffbackup/backups/server2
