Well, I was thinking of something that wrote extended logging to stdout or the 
log file, that would simply be ignored when in production mode. That would 
help when things are not runnning as expected.

Well, this is something I need to think about. In a way, make the preferences 
related to the backup script. ie, imagine that a parent directory in a remote
