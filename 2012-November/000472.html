Thanks Patrik I tried it now and the column sorting works well with 
small directory listings. But with a big listing it takes a really long 
time and Firefox issues several warnings about an unresponsive script 
(jquery.sortelements.js). In the end it finishes okay, but the first 
click of the 'Last Revision' column has put them into ascending date 
order so I then have to click again to get the more useful descending 
date order. Which takes a very long time again.

Apart from this issue I have a perception that this version is slower 
than regular 0.6.3 - but this may not be correct. In general rdiffweb 
(whichever version) gets very slow if it has to load a large directory 
listing. This is understandable but it would be better if it was faster, 
I think many users who aren't used to it would give up long before the 
page had loaded (it can take more than 5 minutes in my experience).

One further idea: I would prefer smaller default text with rdiffweb. The 
large text means that sometimes the most important stuff (version 
details) gets squashed over to the right-hand side of the screen (or 
even off it). See the attached screenshot. This is not a new problem but 
I think a smaller default text would help.

Thanks for your great work, it's really good to see some improvements 
coming to rdiffweb!

Dominic
