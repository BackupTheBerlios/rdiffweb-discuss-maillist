This is fixed in svn. Cherrypy 3 uses two log files, an error file and a 
log file, like Apache. To accommodate this, I've removed the commandline 
parameter and have added two config file settings: accessLogFile and 
configLogFile.

I'll release a new version soon with this fix and the fixes for email 
notifications.
