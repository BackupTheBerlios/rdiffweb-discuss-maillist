Greetings.

 

More updates from me. 

I still can’t check in to subversion (I have tried), so instructions on
how to try my fixes will follow later.

 

All the work I have done is based on the TODO list on the roadmap

http://www.rdiffweb.org/wiki/index.php?title=Roadmap

 

First; What I have changed:

The init script  (as in “service rdiff-web start”)

Now honours stop, start and status. 

Stop and start check whether they have succeeded and report an appropriate
message and exit code

 

rdiff-web-config is now a little more sane.

If an old rdw.conf file exists, its values are taken as the default.

You also have a nice list of what you can do with users existing.

 

Authentication

You now have the option of using PAM (the Unix users/passwords). This is
currently not working on Centos5.2, but works fine on Ubuntu.

(You can still use the internal RDW users on any system).

When PAM is enabled the administrators can’t change other user’s password
(this would violate unix security), but every user can change their own
password.

 

Directory listing

You can now sort the display by Name, size, date or number of old versions.

Just click on the appropriate heading to sort, click again to reverse the
sort.

 

Previous versions

The number of previous versions is now a link to a page that brings up only
the old versions of that file.

Basically, if the popup/flyout is not working on your browser, or is too
long to be useful, just click on “947 Previous Revisions” 

for a page where you can see revision 1 and 948.

 

Annoying change:

I have had to disable SSL; I don’t quite know why, perhaps I have the
wrong version of cherrypy.

 

Technical change:

I have altered the template engine to honour “\&”.

Basically, if a page tries to render “this & that”, the rdw template
engine changes it to “this & that”.

Now, if you try to render “this & that \♥” the template engine
will produce “this & that ♥” 

(hopefully the browser will then correctly show “this & that ♥”)
