Hi,

this problem is hard to reproduce, because there are many ways how to  
mount a share.

Depending on the configuration, rdiff-backup decides the quoting schema.  
In my case it's:

      > Fatal Error: New quoting requirements!
      >
      > The quoting chars this session needs  
'A-Z\x00-\x1f"*/:<>?\\\\|\x7f;' do not match
      > the repository settings '^a-z0-9_ -.' listed in

Since there are escaped characters in the chars_to_quote file, you can  
grab it from here:
http://88.198.44.165/chars_to_quote.gz

If this chars_to_quote file is in the rdiff-backup-data folder, rdiffweb  
will break as described earlier. If it's replaced with the old quoting  
style ('^a-z0-9_ -.') everything is browsable again.

The quoting schema 'A-Z\x00-\x1f"*/:<>?\\\\|\x7f;' looks very strange to  
me - so might be there's something wrong with my configuration .. however,  
I would appreciate it very much if you would have a look a it.
