I have multiple rdiff backup 'locations', created with rdiff-backup  
1.1.5-4 (Debian Etch), and rdiffWeb 0.6.3 was working perfectly.

The backups are on a NAS box, mounted via smbfs (case-insensitive).

Because of smb compatibility I upgraded rdiff-backup to 1.2.1-1  
(Etch-Backport). On the next backup over the existing folders,  
rdiff-backup upgraded to the new quoting style.

 From then on, the upgraded backup locations show up with a size of 0 Byte  
(Last Backup Size), and only the root folder of that backup is accessible  
over the browser. If I try to go deeper, rdiffWeb says "The backup  
location does not exist.".

This occurs even with completely new created backup locations on that  
drive.

However, I can fix it if I revert the "chars_to_quote" file in  
rdiff-backup data folder to the previous version (from 'A-Z-"*/:<>?\\|;'  
to '^A-Za-z0-9_ -.'). If I do so, the backup can be browsed again, but I  
believe any further backups/restores will fail - so I need anonther  
solution for this or I have to go back to rdiff-backup 1.1.5.
