Hello.

 

Where did you get this copy of rdiffweb? The "0.7.0b" code is not officially
added yet.

 

If you want to get the latest "Stable" or "testing" version of rdiff-web,
get it from http://www.rdiffweb.org/wiki/index.php?title=Main_Page#Download

 

If you want the latest "official" development version, you need to get it
from SVN from http://developer.berlios.de/svn/?group_id=5939

 

If you want to stay with the "unofficial" 0.7.0b code (I must confess that I
like the changes it includes), try this:

 

If you run from a root command prompt:

/usr/bin/rdiff-web

You should get an error message that ends with the line:

ImportError: No module named PAM

If you do, there are 2 solutions:

1)      Install PAM. This doesn't work on CentOS or other RedHat
derivatives, so don't bother trying.

2)      Make the following changes by hand:

a.       Edit /usr/lib/python2.4/site-packages/rdiffWeb/page_prefs.py and
remove line 9 "import PAM"

b.      Edit /usr/lib/python2.4/site-packages/rdiffWeb/page_main.py and
remove line 10 "import PAM"

(It may not be "python2.4" on your system if you have upgraded python, but
you should be able to find them)

 

I will make better versions of those two changes in the 0.7.0b code (I
suppose it will be 0.7.1b, then),

But please be aware that the 0.7.x code base is _not_ official until the
code is added to SVN by one of the official developers. 0.7.x is my
modifications that I uploaded to the mailing list for feedback and comment.
It may not even be called 0.7.x if/when it is included.

 

Best of luck, and sorry for the bug.

 

David.
