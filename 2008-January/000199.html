The new testing version of rdiffWeb, 0.6.0, has been released. Notable 
changes since 0.5.1:
* rdiffWeb now uses cherrypy 3. Cherrypy 2.x is not supported.
* sqlite is now used for the user backend. rdiffWeb automatically 
converts mysql and file-based databases into the new format.
* Web-based initial configuration

Upgrading from 0.5.1 involves upgrading cherrypy to version 3.0, and 
installing sqlite's python python bindings (python-pysqlite2 in 
debian-based distros).

As always, bug reports are appreciated.
