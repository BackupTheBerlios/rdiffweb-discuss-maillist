It's time for another release soon. There are three significant changes 
since 0.5:
* Move to cherrypy version 3.0. This means that rdiffWeb is no longer 
compatible with cherrypy 2.x. This was necessary for maintainability and 
HTTPS support.
* Moving to a sqlite database for user information. This is designed to 
make setup easier. No user action other than installing the sqlite 
python bindings is required for this.
* Adding a web-based configuration for first-time setup. Again, this is 
designed to make setup easier.

Although no user action is required for the new version of rdiffWeb 
itself, you will need to download and install cherrypy 3.0, as well as 
pysqlite.

I would welcome testing on the current code, particularly relating to 
the new sqlite user database.
