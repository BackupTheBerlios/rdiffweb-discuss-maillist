The heavens stretched forth and Martin Fisher decreed the following on
or about 12/18/2007 06:28 AM:
> I am trying to start using rdiffweb (v useful, thank you!) with
> rdiff-backup. Following a previous thread
> 
>> # mysqladmin -u root -h localhost password "whatever_you_want" (set
>>  mysql root password)
>> 
>> If you don't mind having rdiffWeb use the mysql root password, then
>> all you need to do at this point is re-run the rdiff-web-config
>> script, and tell it to use "localhost" for the host, "root" for the
>> username, and the password you gave it.
> 
> I have tried to set up mysql. I have installed it with Ubuntu (7.1)
> but receive the following message:
> 
> $ mysqladmin -u root -h localhost password "testpwd1"
> 
> mysqladmin: connect to server at 'localhost' failed error: 'Can't
> connect to local MySQL server through socket
> '/var/run/mysqld/mysqld.sock' (2)' Check that mysqld is running and
> that the socket: '/var/run/mysqld/mysqld.sock' exists!
> 
> I have never used mysql before and I'm sure the solution is simple,
> like starting mysql properly. Could anybody more knowledgeable kindly
> enlighten me?

Yeah, that looks like mysql isn't started.  Knowing that error is 
crucial because rdiffweb will spit out something similar if you try to 
run it with out mysqld running as well.

I'm a Fedora man, but I *think* the way you start it on Ubuntu is:

   sudo /etc/init.d/mysql start

The graphical way is to do System->Services->Database server?  I think 
after installation through Synaptic or whatever, Ubuntu will set it up 
to start after booting.  If not, I think that System->Services probably 
has the way to set that as well.
