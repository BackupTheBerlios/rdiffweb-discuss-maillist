This is currently a problem with the init.d script -- it doesn't pass 
back errors if the daemon fails to start!

To check for errors on startup, run rdiff-web -d on the commandline. It 
should give errors. If that doesn't give enough information, post the 
output to this list, and we'll try to help you.
