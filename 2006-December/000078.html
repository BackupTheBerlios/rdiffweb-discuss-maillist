I'd have a strong preference for only tweaking the css, for several reasons:
* The templating system is very strict about keyword replacement -- a 
keyword specified in the template that does not exist in the parameters 
passed will result in a server exception 500. Having many different 
themes would put a big burden on maintainers, since every time they 
change a template, they need to change all the other "themed" templates.
* Conceptually, that is what css is for -- altering the display of the 
data. I realize that it doesn't always work out that way in the real 
world, but it's a goal to shoot for.
* It makes theming easier to add. If we would ever make it a user 
preference, it would be trivial to support, because we would simply need 
to change the path of the css file.

Of all these reasons, the first is the most important, IMO. Thoughts?
