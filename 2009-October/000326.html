I haven't had a chance to take a look at the code, but from a functional 
perspective:
* The PAM authentication setting should be site-wide, rather than 
user-specific.
* I would strongly prefer that if PAM is enabled, changing the user 
password changes the system password. (If PAM doesn't allow this, you 
could use a script that sets the user password, like samba does.)

Thanks for working on this! This stuff will be really nice.

> (or if you already have it installed and know what you are doing, copy

> /usr/local/lib/python2.6/dist-packages/rdiffWeb as appropriate)
>
>  
>
> PAM adds the requirement for python-pam 0.4.2 (sudo apt-get install 
> python-pam)
>
>  
>
> Changes:
>
> when using the init script (or service rdiff-web <command>),
>
> "start" - confirms that the start worked, and reports the PID
>
> "stop" - confirms the stop succeeds.
>
> "status" - returns the pid if running.
>
>  
>
> Admin page:
>
> Creating or editing users offers a "Authenticate with PAM?" checkbox.
>
>  
>
> Preferences page:
>
> If current user is set to use PAM, a message notes that changing the 
> password on this page has no effect
>
> In fact, it changes the user's password in rdiff-web's DB, but since 
> the shadow password is used, this won't have any effect until an 
> administrator switches the user to non-PAM.
>
>  
>
> If you have an rdiff-web user who does not have a unix system account, 
> setting them to use PM effectively locks the account.
