Greetings, all.

 

I see in the TODO list at 

http://www.rdiffweb.org/wiki/index.php?title=Roadmap 

that init script improvements , shadow passwords (PAM) and logging are High
priority improvements.

 

I have improved teh init script and added PAM support.

 

I have not done Logging because I'd like to find out what people want logged
before I do anything about it. I'll watch this list.

 

I can't/don't know how/won't commit to svn so to get my changes, do this:

Check out the latest version from "trunk" in svn.

Then extract the attached archive over the top of that _before_ installing
it.

(or if you already have it installed and know what you are doing, copy the
init script  from the archive to /etc/init.d and the others to
/usr/local/lib/python2.6/dist-packages/rdiffWeb as appropriate)

 

PAM adds the requirement for python-pam 0.4.2 (sudo apt-get install
python-pam)

 

Changes:

when using the init script (or service rdiff-web <command>), 

"start" - confirms that the start worked, and reports the PID

"stop" - confirms the stop succeeds.

"status" - returns the pid if running.

 

Admin page:

Creating or editing users offers a "Authenticate with PAM?" checkbox.

 

Preferences page:

If current user is set to use PAM, a message notes that changing the
password on this page has no effect 

In fact, it changes the user's password in rdiff-web's DB, but since the
shadow password is used, this won't have any effect until an administrator
switches the user to non-PAM.

 

If you have an rdiff-web user who does not have a unix system account,
setting them to use PM effectively locks the account.

 

I hope the attachment makes it to the mailing list.

 

Wish me luck!
