The documentation on the wiki states that the UseHttps configuration 
setting is required so that rdiffWeb writes URLs with the correct scheme 
(see http://www.rdiffweb.org/wiki/index.php?title=UsingHttps). First of 
all, now that rdiffWeb uses its own authentication filter, is this URL 
rewriting still necessary?

If it is, it may still be possible to eliminate configuration setting 
while keeping the filter. I recently noticed that the URI specification 
(http://www.gbiv.com/protocols/uri/rfc/rfc3986.html#reference-examples) 
describes a syntax that allows URIs to be described relative to the 
scheme. If you have a link to //www.rdiffweb.org/wiki/, it will resolve 
to https://www.rdiffweb.org/wiki/ on a secure site and to 
http://www.rdiffweb.org/wiki/ on a non-secure site. Rather than 
rewriting URLs to use the https scheme, the filter could rewrite all 
URLs to use no scheme (assuming that CherryPy handles it okay).

These changes would make integration with Stunnel seamless.
