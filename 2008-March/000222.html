AJ Kertis wrote:
> Hi,
>
> I have successfully setup rdiffweb to work with rdiff-backup but I have a
> few issues and wasn't sure if anyone knew a solution. I can't seem to
> restore a backup from the web interface. After i click on a backup and then
> restore folder it does ask me the revision I want to backup but after that
> it just sits there and eventually I have to stop it and then restart the
> rdiffweb daemon. Also in ps -ef there is a process but it seems it is
> restoring the backup to /tmp and is there a way so it will restore it to the
> actual machine that was backed up? Is this a misconfiguration in some way
> that i've done? 
rdiffWeb restores the data to a temporary location, then zips it up and 
lets you download it as a zip file. If it takes a long time after you 
select a revision to restore to then you are probably restoring a large 
amount of data.

> Also, what are the parameters to set up email alerts I was
> on the wiki and saw the parameters for the smtp host and the from address
> but is there a parameter for where to send the emails? Any help would be
> great!
The email address to send to is located in the user settings, which are 
accessed via the web interface.
