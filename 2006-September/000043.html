Couple of things:
* First, we wanted to do exactly the same thing, so we used an RSS feed. 
You can click on the "Feed" icon on the bottom of the status page for 
the link. I use this in Thunderbird (both 1.5 and 2.0 support 
authenticated RSS feeds pretty well.) I like this better than a daily 
email, since it checks every 30 minutes, so I find out about failed 
backups sooner.
* rdiffWeb supports HTTP authentication, but only for the RSS feed.  
(This is because it has a minor security hole: using HTTP authentication 
leaves the session open for an hour.) To enable it for everything, 
you'll need to change the main rdiff-web script. It's in the config 
section close to the bottom; I think it should be fairly self-explanatory.
* If you really do want to script pulling down the status page, you 
should be able to do it with curl. I would recommend googling for (perl) 
scripts to download stuff from allofmp3.com, since the technique is 
similar. You'll want to set up curl to store cookies, then request the 
status page, use post-data, then use those cookies to do a second 
request to actually get the status page.
